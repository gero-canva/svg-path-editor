# Plan: Wrap Angular SPA In Tauri For macOS

## Scope

Build a Tauri v2 desktop wrapper for the existing Angular 17 SPA that can produce an unsigned local macOS app bundle. Add native macOS menus and accelerators for standard app commands while preserving the existing in-webview SVG editing shortcuts.

Chosen constraints:

- Use Tauri v2.
- Target unsigned local macOS builds first; no signing or notarization setup.
- Keep existing browser-style open/save/export/share flows; do not add native file dialogs in this pass.
- Keep single-letter SVG command shortcuts in the Angular webview rather than making them global/native accelerators.
- Remove Google Analytics from the app.
- Vendor Google-hosted frontend assets locally so the Tauri app loads fonts and Material Icons from bundled files.

## Key Existing Code

- `package.json`: npm scripts and Angular dependencies.
- `package-lock.json`: update when adding Tauri npm packages.
- `angular.json`: Angular build outputs to `dist/svg-path-editor`.
- `src/index.html`: already uses `<base href=".">`, which is suitable for Tauri bundled assets; currently loads Roboto, Material Icons, and Google Analytics from Google-hosted URLs.
- `src/styles.scss` or a new local stylesheet: add local `@font-face` declarations for vendored fonts/icons.
- `src/app/app.component.ts`: current shortcut handling in `onKeyDown`, plus app actions such as undo/redo/delete/zoom/sidebar/path open.
- `src/app/app.component.html`: toolbar buttons and child components for open/save/export/share/upload.
- `src/app/app.module.ts`: production service worker registration.
- `src/app/constants/keyboard.const.ts`: existing key constants.
- `.gitignore`: should ignore Rust/Tauri build output.
- `README.md`: document Tauri usage and desktop shortcut behavior.

## Implementation Steps

1. Add Tauri dependencies and scripts.
   - Add `@tauri-apps/cli` as a dev dependency and `@tauri-apps/api` as a dependency.
   - Add scripts: `tauri`, `tauri:dev`, and `tauri:build`.
   - Keep `build-prod` unchanged for GitHub Pages; Tauri should use the normal Angular build output, not `/svg-path-editor/` base href.

2. Add a Tauri v2 project scaffold.
   - Add `src-tauri/Cargo.toml` with `tauri = "2"` and `tauri-build = "2"`.
   - Add `src-tauri/build.rs` using `tauri_build::build()`.
   - Add `src-tauri/src/main.rs` for the desktop shell.
   - Add `src-tauri/tauri.conf.json` with `beforeDevCommand: "npm run start"`, `devUrl: "http://localhost:4200"`, `beforeBuildCommand: "npm run build"`, and `frontendDist: "../dist/svg-path-editor"`.
   - Use product name `SVG Path Editor`, identifier `com.local.svgpatheditor`, and a `main` window around `1280x900`.
   - Configure unsigned bundle output for `app` and `dmg`; do not configure Apple signing or notarization.

3. Add Tauri capabilities and icons.
   - Add `src-tauri/capabilities/default.json` for the `main` window with minimal core/event permissions needed by the frontend event listener.
   - Generate `src-tauri/icons/*` from `src/assets/icons/icon-512x512.png` using the Tauri CLI after dependencies are installed.
   - Add the generated icon paths to `bundle.icon` in `tauri.conf.json`.
   - Add `/src-tauri/target` to `.gitignore`; commit source/config/icons/locks but not Rust build artifacts.

4. Vendor external frontend assets and remove analytics.
   - Remove the Google Analytics script block and `googletagmanager.com` script from `src/index.html`.
   - Remove Google Fonts and Material Icons remote stylesheet links from `src/index.html`.
   - Download Roboto font files for weights `300`, `400`, and `500` and commit them under `src/assets/fonts/roboto/`.
   - Download the Material Icons ligature font used by Angular Material `mat-icon` text content and commit it under `src/assets/fonts/material-icons/`.
   - Add local `@font-face` declarations, preferably in `src/styles.scss`, using relative asset URLs such as `assets/fonts/roboto/...woff2` and `assets/fonts/material-icons/...woff2`.
   - Preserve the expected `font-family: 'Material Icons'` ligature behavior so existing usages like `<mat-icon>save</mat-icon>` continue to render without template changes.
   - Keep existing SVG icons registered through `MatIconRegistry` unchanged.

5. Implement native macOS menus in Rust.
   - Build top-level macOS submenus: app, File, Edit, View, Window, Help.
   - Use native/predefined menu items where appropriate for app/window behavior.
   - Use custom menu item IDs for editor actions and emit a `native-command` event to the `main` webview.
   - Planned custom commands:
     - `new-path` with `CmdOrCtrl+N`.
     - `open-paths` with `CmdOrCtrl+O`.
     - `save-path` with `CmdOrCtrl+S`.
     - `export-svg` with `CmdOrCtrl+E`.
     - `undo` with `CmdOrCtrl+Z`.
     - `redo` with `CmdOrCtrl+Shift+Z`.
     - `delete-selection` as a menu item without a native accelerator; `Delete`/`Backspace` stay webview-handled.
     - `zoom-in` with `CmdOrCtrl+=`.
     - `zoom-out` with `CmdOrCtrl+-`.
     - `zoom-fit` with `CmdOrCtrl+0`.
     - `toggle-sidebar` with `CmdOrCtrl+B`.

6. Add an Angular command dispatcher.
   - Add a small `EditorCommand` union in `src/app/app.component.ts` matching the native command payloads.
   - Add `executeEditorCommand(command: EditorCommand)` and route native menu events through the same methods used by existing buttons.
   - Add `@ViewChild` references for `OpenComponent`, `SaveComponent`, `ExportComponent`, and `ShareComponent` so native menu commands can open existing Angular dialogs without changing the file UX.
   - Add named methods for inline zoom actions, e.g. `zoomIn()` and `zoomOut()`, so menu events and template buttons share the same logic.
   - Keep the single-letter SVG insert/convert shortcuts, `Escape`, `Delete`/`Backspace`, and drag-time `Ctrl` behavior in the webview.

7. Bridge Tauri events into Angular safely.
   - Dynamically import `listen` from `@tauri-apps/api/event` only when running inside Tauri, detected with `__TAURI_INTERNALS__ in window`.
   - Store and call the returned unlisten function in `ngOnDestroy`.
   - Wrap native event handling in `NgZone.run(...)` so Angular change detection sees menu-triggered state updates.

8. Avoid shortcut double execution and text-input regressions.
   - Add a shared `isEditableTarget(...)` guard covering `INPUT`, `TEXTAREA`, `SELECT`, and `contenteditable` elements.
   - Preserve browser behavior outside Tauri.
   - In Tauri, let native menu accelerators own standard commands such as undo/redo/new/open/save/export/zoom/sidebar, while the webview handler continues to own SVG editing shortcuts.
   - For native `undo`/`redo`, ignore the command when focus is in an editable field so text editing shortcuts still work normally.

9. Disable service worker registration inside Tauri.
   - Update `src/app/app.module.ts` so `ServiceWorkerModule.register(...)` remains enabled for browser production builds but is disabled when running inside Tauri.
   - This avoids custom-protocol/service-worker cache issues without changing the web deployment behavior.

10. Update docs.
   - Add README instructions for `npm run tauri:dev` and `npm run tauri:build`.
    - Note that macOS builds are unsigned and may require right-click Open or Gatekeeper approval.
    - Document that standard commands are native menu accelerators in the desktop app, while SVG letter shortcuts remain in-canvas/webview shortcuts.
    - Document that fonts/icons are bundled locally and analytics has been removed.

## Verification

Run what is possible in this workspace:

- `npm install`
- `npm run build`
- `npm run lint`
- `npm test -- --watch=false` if the existing Karma setup can run headlessly here.
- `npm run tauri:dev` if Rust/Tauri desktop dependencies are available in the environment.
- `cargo check` from `src-tauri` if Rust dependencies can be resolved.

Manual desktop checks on macOS latest:

- `npm run tauri:dev` loads the Angular app in a Tauri window.
- `npm run tauri:build` produces an unsigned `.app` and `.dmg` under `src-tauri/target/release/bundle`.
- Existing webview shortcuts still work: SVG command letters, shifted conversion letters, `Escape`, `Delete`/`Backspace`, and `Ctrl` while dragging.
- Native menu accelerators work once, not twice: `Cmd+Z`, `Cmd+Shift+Z`, `Cmd+N`, `Cmd+O`, `Cmd+S`, `Cmd+E`, `Cmd+=`, `Cmd+-`, `Cmd+0`, `Cmd+B`.
- Text input behavior is preserved: `Cmd+Z`/`Cmd+Shift+Z` in path or numeric fields should affect text editing, not the path history.
- Existing browser flows still work in Tauri: open saved path dialog, save path dialog, export/download, share copy, and image upload.
- Run or inspect the built Tauri app with the network disabled and confirm Roboto text and Material Icons render correctly.
- Check the Tauri webview console for service worker, CSP, missing local font/icon asset, and unexpected external request errors.
- Confirm there are no remaining `googletagmanager.com`, `gtag`, `fonts.googleapis.com`, or `fonts.gstatic.com` references in `src/`.

## Risks

- This Linux workspace cannot fully validate macOS bundle behavior; final `.app`/`.dmg` verification must happen on macOS.
- Tauri menu/Rust APIs should be verified against the installed Tauri v2 version during implementation.
- Vendored font file URLs may change upstream; during implementation, fetch the current CSS once, commit the resulting `.woff2` assets, and ensure no runtime Google font requests remain.
- Browser download/clipboard behavior may differ slightly in a webview; native file dialogs are intentionally out of scope for this pass.
